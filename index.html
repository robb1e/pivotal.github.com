Pivotal Labs ♥ developers

Here are a collection of our favorite Open Source projects including those from ourselves, our clients, our developers and our friends.

  • Jasmine

    Jasmine is a behavior-driven development framework for testing JavaScript code. It does not depend on any other JavaScript frameworks. It does not require a DOM. And it has a clean, obvious syntax so that you can easily write tests.

    Find out more

  • Lobot

    Lando Calrissian relies on Lobot to keep Cloud City afloat, and now you can rely on Lobot to get your continuous integration server running in the cloud. Lobot is a gem that will help you spin-up, bootstrap, and install Jenkins CI for your Rails app on Amazon EC2.

    Find out more

  • Cedar

    BDD-style testing using Objective-C

    Find out more

  • Workstation

    Pivotal Workstation: A Repeatable, Documented, Decomposable, Shareable and Iterative OSX (ruby) Development Environment

    Find out more

  • License Finder

    With bundler it's easy for your project to depend on many gems. This decomposition is nice, but managing licenses becomes difficult. This tool gathers info about the licenses of the gems in your project.

    Find out more

  • Project Monitor

    ProjectMonitor is a CI display aggregator. It displays the status of multiple Continuous Integration builds on a single web page. The intent is that you display the page on a big screen monitor or TV so that the status of all your projects' builds are highly visible/glanceable (a "Big Visible Chart")

    Find out more

  • Git Scripts

    These scripts are helpers for managing developer workflow when using git repos hosted on GitHub

    Find out more

  • Postgres Search (pg_search)

    PgSearch builds named scopes that take advantage of PostgreSQLs full text search

    Find out more

  • Quacky

    Ruby doubles and expectations that conform to duck types.

    Find out more

  • Whiteboard

    Whiteboard is an app which aims to increase the effectiveness of office wide standups, and increase communication with the technical community by sharing what we learn with the outside world. It does this by making two things easy - emailing a summary of standup to everyone in the company and by creating a blog post of the items which are deemed of public interest.

    Find out more
